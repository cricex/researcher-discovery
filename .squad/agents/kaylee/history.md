## Learnings

- **Agent HTTP contract types** live in `src/specs/agent-contract.ts`. These define the wire-format shapes (AgentRequest, AgentContractResponse, AgentErrorResponse, AgentHealthResponse) and AgentErrorCode enum. Property names are camelCase; the HTTP client layer is responsible for snake_case mapping. Spec source: `specs/001-orchestration-agent/contracts/agent-contract-spec.md`.
- The existing specs style uses JSDoc on every exported type/interface, module-level doc comments, and `.js` extensions in barrel exports. Follow this pattern for any new spec files.
- **Team Updates (Phase 1 Setup):** River pivoted IntentCategory to research domain (EXPERTISE_DISCOVERY, RESEARCH_OUTPUT, COLLABORATION_INSIGHT, POLICY_COMPLIANCE, GENERAL) with new `context` field on ClassifiedIntent. Wash extended response.ts with Citation, OrchestrationResult, AgentErrorEntry — all 15 tests passing. New shared contract contracts are aligned for orchestrator core implementation.
- **Agent endpoint config** lives in `src/agents/endpoints.ts`. `AgentEndpointConfig` interface defines connection details (id, name, url, healthUrl, timeoutMs, enabled). `DEFAULT_AGENT_ENDPOINTS` is a `Map<string, AgentEndpointConfig>` with the three known agents (expertise_discovery :5001, research_output :5002, policy_compliance :5003).
- **HTTP agent client** lives in `src/agents/client.ts`. `HttpAgentClient` class uses native `fetch` with `AbortController` timeouts. It maps camelCase TypeScript types ↔ snake_case wire format via internal `toWireRequest`/`fromWireResponse`/`fromWireError` functions. `checkHealth` never throws — returns `status: 'unhealthy'` on failure. `callAgent` throws typed `AgentErrorResponse` on timeout, HTTP errors, and network failures.
- **Team Updates (Phase 2 Foundation):** River created `src/orchestrator/classifier/routing-rules.ts` with Map<IntentCategory, string[]> for four domain categories (EXPERTISE_DISCOVERY, RESEARCH_OUTPUT, COLLABORATION_INSIGHT, POLICY_COMPLIANCE). Zoe expanded test helpers with createMockAgentResponse, createMockAgentRequest, createStubHttpAgent factories and updated registry.test.ts for new enum values — all 15 tests passing. Phase 2 foundation infrastructure complete.
- **FastAPI backend** created at `src/api/main.py`. Wraps the Python `Orchestrator` class from `src/orchestrator/orchestrator.py`. Uses `sys.path` manipulation to resolve the import from `src/api/` → `src/orchestrator/`. Constructor: `Orchestrator(use_mock_agents=True)`. Key method: `route_query(query)` returns `OrchestrationResult` dataclass with `.response`, `.agents_invoked`, `.citations`, `.confidence`, `.reasoning`, `.metadata`. Cleanup via `close()`. Modernized for Pydantic v2 (`model_dump()`) and FastAPI lifespan context manager. Endpoints: `/`, `/health`, `POST /api/v1/query`, `GET /api/v1/session/{id}`, `DELETE /api/v1/session/{id}`, `GET /api/v1/agents`, `WS /ws/orchestration`. CORS allows ports 3000 and 5173.
- **Wave 1 Cross-Team (2026-03-05T20:30):** FastAPI backend integration with Wash's React frontend (ports 3000 Vite dev, 5173 CORS). Kaylee's B1 is foundational for Wash's B2/B3 and Mal's B6/B7 integration docs. River's T010 tests use routing-rules from T006 (Kaylee baseline from Phase 2). All services orchestrated in parallel via dual-track Wave 1 delivery.
- **HttpAgent adapter** (T014) lives in `src/agents/http-agent.ts`. Implements `Agent` by wrapping `AgentEndpointConfig` + `HttpAgentClient`. Uses a `CATEGORY_MAP` to derive `IntentCategory` from config id. `canHandle` matches intent category; `execute` builds an `AgentRequest`, calls `callAgent()`, and maps `AgentContractResponse` → `AgentResponse` with citations in metadata. Error path returns `status: 'error'` with the message extracted from the thrown `AgentErrorResponse`.
- **AgentRegistry convenience methods** (T015): `registerEndpoint(config, client?)` creates an `HttpAgent` and registers it in one call. `registerDefaults()` iterates `DEFAULT_AGENT_ENDPOINTS` and registers all agents using a shared `HttpAgentClient`, skipping any already-registered ids. Both added to `src/agents/registry.ts`.
- **ResponseAggregator extracted (B5):** `src/orchestrator/aggregator.py` is the dedicated aggregation module. `ResponseAggregator` class owns `aggregate_responses()` → `AggregatedResult` dataclass (sections, citations, confidence, metadata, agents_used). Domain sectioning via `_section_by_domain()` groups output into Expertise/Research/Collaboration/Policy. Citation dedup is order-preserving exact-match. Confidence is weighted average of per-agent confidence × success ratio. `render_markdown()` static method converts `AggregatedResult` → flat markdown string. `orchestrator.py` now imports and delegates to this class instead of inline `_aggregate_responses`. The FastAPI layer (`src/api/main.py`) needed no changes — it only imports `Orchestrator` and `OrchestrationResult`.
